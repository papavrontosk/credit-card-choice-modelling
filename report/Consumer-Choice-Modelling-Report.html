Question 1 — Utility Function Specification

Choice Set

Each individual i faces a choice set of J = 5 alternatives: alternatives 1–4 are distinct credit-card products, and alternative 5 is the opt-out (no bank card chosen). The dataset confirms that alternative 5 always carries fee = 0, rate = 0, and rewards = 0.

Systematic Utility Components

Under Random Utility Maximization, individual i chooses alternative j if Uij > Uik for all k ≠ j, where:

Uij = Vij + εij

The systematic part Vij for credit-card alternatives j = 1, 2, 3, 4 is:

Vij = αj + βfee·feeij + βrate·rateij + βrewards·rewardsij + γincome,j·incomei + γage,j·agei

For the opt-out alternative j = 5:

Vi5 = α5 + γincome,5·incomei + γage,5·agei

Identification Normalisation

Alternative 1 serves as the base category: α1 = 0, γincome,1 = 0, γage,1 = 0. This normalisation removes the location indeterminacy of the MNL model.

Justification of Generic vs Alternative-Specific Parameters

Generic coefficients (β): fee, rate, rewards Card attributes are common across all alternatives they appear in. A consumer responds to cost regardless of which card carries it — the disutility of paying €1 in fees is the same whether it is card 2 or card 4. Using a single β for each attribute imposes preference homogeneity over the attribute itself (not over the alternatives), which is the standard and theoretically justified approach for product attributes in the discrete-choice literature.
Alternative-specific constants (αj) Each card may have unobserved attributes (brand reputation, credit limit, insurance, acceptance network). The ASC captures the average utility of choosing alternative j over the base, net of the observed attributes. The opt-out ASC (α5) captures the average utility of not having any bank card.
Alternative-specific sociodemographic interactions (γincome,j, γage,j) Income and age are individual-level variables that are constant across alternatives for a given person. They therefore cannot enter as generic terms — doing so would add the same value to all utilities and cancel out. They enter as alternative-specific interactions: γincome,j·incomei measures how a higher income differentially shifts utility towards alternative j relative to the base. This is the correct specification for individual characteristics in unlabelled choice sets. (They are fixed for each person but vary in their effect on preference for each alternative.)

Question 2 — MNL Estimation via Maximum Likelihood

Data Loading & Preparation

R #| echo: true | #| eval: true
# ── Load data ─────────────────────────────────────────────────────────
df <- read.csv("credit.csv")
n  <- nrow(df)

# Alternative attributes (N × 5 matrices)
fees    <- as.matrix(df[, paste0("fee_", 1:5)])
rates   <- as.matrix(df[, paste0("rate_", 1:5)])
rewards <- as.matrix(df[, paste0("rewards_", 1:5)])
income  <- df$income / 1000   # scale to thousands €
age     <- df$age
choice  <- df$choice         # 1..5
balance <- df$balance

# ── Log-likelihood function ────────────────────────────────────────────
# params[1]  = beta_fee
# params[2]  = beta_rate
# params[3]  = beta_rewards
# params[4:7]  = alpha_2..alpha_5  (alpha_1 = 0, normalisation)
# params[8:11] = gamma_income_2..5 (gamma_income_1 = 0)
# params[12:15]= gamma_age_2..5    (gamma_age_1 = 0)

log_lik <- function(params) {
  bf  <- params[1]; br <- params[2]; brw <- params[3]
  alp <- c(0, params[4:7])
  gi  <- c(0, params[8:11])
  ga  <- c(0, params[12:15])

  # Systematic utility (n × 5)
  V <- bf * fees + br * rates + brw * rewards +
       matrix(alp, n, 5, byrow = TRUE) +
       income %o% gi +
       age    %o% ga

  # Numerically stable softmax
  Vmax <- apply(V, 1, max)
  eV   <- exp(V - Vmax)
  P    <- eV / rowSums(eV)

  chosen <- P[cbind(seq_len(n), choice)]
  -sum(log(chosen + 1e-300))
}

# ── Optimisation ──────────────────────────────────────────────────────
x0  <- rep(0, 15)
res <- optim(x0, log_lik, method = "L-BFGS-B",
              control = list(maxit = 50000, factr = 1e-12),
              hessian = TRUE)

params_hat <- res$par
se         <- sqrt(diag(solve(res$hessian)))
t_stat     <- params_hat / se
p_val      <- 2 * pnorm(-abs(t_stat))

cat("Log-likelihood:", -res$value, "\n")
cat("Converged    :", res$convergence == 0, "\n")
Log-likelihood: -703.587 Converged : TRUE

Own-Alternative Marginal Effects — Fee

R #| echo: true | #| eval: true
# For continuous attribute x in alt j:
# ∂P_j/∂x_j = β_x · P_j · (1 − P_j)
# Average across all 4 card alternatives and all observations
compute_probs <- function(params, fees., rates., rewards.) {
  bf<-params[1]; br<-params[2]; brw<-params[3]
  alp <- c(0,params[4:7]); gi <- c(0,params[8:11]); ga <- c(0,params[12:15])
  V <- bf*fees. + br*rates. + brw*rewards. +
       matrix(alp,n,5,byrow=TRUE) + income%o%gi + age%o%ga
  Vm <- apply(V,1,max); eV <- exp(V-Vm)
  eV / rowSums(eV)
}
P <- compute_probs(params_hat, fees, rates, rewards)

me_fee  <- mean(params_hat[1] * P[,1:4] * (1 - P[,1:4]))
me_rate <- mean(params_hat[2] * P[,1:4] * (1 - P[,1:4]))

cat(sprintf("Mean own ME, fee  : %.5f  (Δ prob per €1 increase)\n", me_fee))
cat(sprintf("Mean own ME, rate : %.4f  (Δ prob per unit increase in rate)\n", me_rate))

Interpretation — Sensitivity to Fees and Rates

Customers are more than four times as sensitive to interest rates as to annual fees in elasticity terms. A 1% relative increase in the annual rate reduces the probability of choosing a card by about 3.31%, while a 1% relative increase in the annual fee reduces it by only 0.67%. In absolute terms, a €1 increase in the annual fee reduces choice probability by 0.15 pp, while a 1 pp increase in the rate (e.g. from 15% to 16%) reduces it by 0.24 pp. Customers are willing to pay €16 per year to have the rate reduced by 1 percentage point. This suggests the bank should prioritise keeping rates competitive above all else.

2b — Value of Rewards Programmes

R #| echo: true | #| eval: true
# Discrete change: all 4 cards go from rewards=0 to rewards=1
# ΔP_j = P_j(rewards=1) − P_j(rewards=0), averaged over j=1..4 and i
rewards_0 <- rewards; rewards_0[,1:4] <- 0
rewards_1 <- rewards; rewards_1[,1:4] <- 1
P0 <- compute_probs(params_hat, fees, rates, rewards_0)
P1 <- compute_probs(params_hat, fees, rates, rewards_1)
dc_rewards <- mean(P1[,1:4] - P0[,1:4])
cat(sprintf("Mean discrete change in card-choice prob (rewards 0→1): %.4f\n", dc_rewards))
cat(sprintf("WTP for rewards programme: €%.2f / year\n", wtp_rewards))
Mean discrete change in card-choice prob (rewards 0→1): 0.0043 WTP for rewards programme: €93.28 / year
Interpretation — Value of Rewards Programmes

The rewards programme has a strongly significant positive effect on choice (β = 1.491, t = 12.85). When all cards switch from no-rewards to rewards, the average probability of choosing each card increases by 0.43 pp. More tellingly, customers are willing to pay €93.28 per year for a rewards programme — nearly six times the willingness to pay for a 1-pp rate reduction. The rewards coefficient has the largest WTP of any attribute, strongly suggesting the bank should maintain and emphasise rewards programmes as a key differentiator.

Sociodemographic Findings

Income and Age Effects Income and age have no statistically significant differential effect on preferences among the four card alternatives (γ coefficients for j=2,3,4 are all insignificant). However, both variables strongly determine the probability of opting out: a higher income significantly reduces the probability of opting out (γincome,5 = −0.996, p<0.001), while older individuals are significantly more likely to opt out (γage,5 = 0.570, p<0.001). In other words: wealthier customers are more likely to hold a credit card; older customers are more likely to decline entirely.

Question 3 — New Card Simulation

The bank proposes replacing all four existing cards with a single new card: fee = €70/year, rate = 20%, rewards programme = yes. Only two alternatives remain in the choice set: the new card and the opt-out (alt 5).

R #| echo: true | #| eval: true
# New card attributes (constant across all individuals)
fee_new     <- 70
rate_new    <- 0.20
rewards_new <- 1

# Unpack parameters
bf <- params_hat[1]; br <- params_hat[2]; brw <- params_hat[3]
alp5 <- params_hat[7]                  # alpha_5
gi5  <- params_hat[11]                 # gamma_income_5
ga5  <- params_hat[15]                 # gamma_age_5

# Systematic utilities (alt 1 = new card; alpha_1=0 by normalisation)
V_new <- bf * fee_new + br * rate_new + brw * rewards_new  # scalar
V_5   <- alp5 + gi5 * income + ga5 * age                   # vector n

# Choice probabilities (numerically stable)
Vmax  <- pmax(V_new, V_5)

cat(sprintf("Share staying with bank (new card): %.1f%%\n", 100*mean(P_new)))

Question 4 — Interest Rate Cap Policy (10% APRC Ceiling)

The government imposes a cap: all credit-card interest rates (APRC) are capped at 10%. Cards currently at rates above 10% are re-priced to exactly 10%; those at or below 10% are unaffected. The sample of 1,000 observations is treated as representative of the bank's 1 million customers.

Interpretation — Bank Revenue Effect The 10% rate cap reduces the bank's annual interest income by approximately €10.8 million (−12.7%) across its 1 million customers. Two channels drive this: (1) a direct mechanical effect — revenue from customers who remain in their current card falls because the applicable rate drops from its current level (15%–25% for affected cards) to 10%; (2) a behavioural effect — lower rates increase the attractiveness of credit cards, slightly raising card-holding probabilities (reducing opt-out), which partially offsets the revenue decline. The net result is a significant revenue hit dominated by the direct price compression.

4b — Consumer Welfare Effect (Logsum)

R #| echo: true | #| eval: true
# ── Logsum-based welfare measure (Small & Rosen 1981) ─────────────────
# CS = (1 / −β_fee) · log(Σ_j exp(V_ij))
# ΔCS = (1 / −β_fee) · [logsum_after − logsum_before]
# β_fee converts utility units to euros (marginal utility of income proxy)

logsum_fn <- function(fees., rates., rewards.) {
  bf<-params_hat[1]; br<-params_hat[2]; brw<-params_hat[3]
  alp <- c(0,params_hat[4:7]); gi <- c(0,params_hat[8:11]); ga <- c(0,params_hat[12:15])
  V   <- bf*fees. + br*rates. + brw*rewards. +
         matrix(alp,n,5,byrow=TRUE) + income%o%gi + age%o%ga
  Vm  <- apply(V,1,max)
  log(rowSums(exp(V-Vm))) + Vm
}

ls_before <- logsum_fn(fees, rates,        rewards)
ls_after  <- logsum_fn(fees, rates_capped, rewards)

# Compensating variation (€ per customer)
# Divide by (−β_fee) to convert utils → euros
cv_per_customer <- (ls_after - ls_before) / (-params_hat[1])
cv_total        <- mean(cv_per_customer) * 1e6

cat(sprintf("Mean CV per customer (€)       : %.2f\n", mean(cv_per_customer)))
cat(sprintf("Total welfare gain (1M customers): €%s\n",
            format(round(cv_total), big.mark=",")))

Interpretation — Consumer Welfare Effect

The logsum measure captures the welfare gain from an improved choice set (lower rates expand utility). The rate cap generates an average compensating variation of +€46.14 per customer per year, or +€46.1 million in total across the 1 million customers. This represents the maximum amount each customer would be willing to pay (as a lump sum) to have the cap in place — equivalently, it is the additional utility from having access to cheaper credit, expressed in monetary terms.

Note: this welfare calculation excludes the direct savings on interest payments (the difference in interest charged to those who continue to carry balances at the lower rate), as instructed. Including those savings would make the consumer-welfare gain substantially larger. As stated, the welfare gain (€46.1M) substantially exceeds the bank's revenue loss (€10.8M) from the non-interest-payment channel alone, implying a net positive social surplus from the policy.
